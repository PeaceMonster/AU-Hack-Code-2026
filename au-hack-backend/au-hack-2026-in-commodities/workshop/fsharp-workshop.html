F# Core Concepts

Let bindings & immutability

let x = 42              // immutable

let mutable counter = 0   // mutable — avoid if possible

Functions

Functions are defined with let, just like values — because functions are values:

let add x y = x + y
let result = add 3 5        // 8

// Partial application: "configure" a function
let addFive = add 5
addFive 3                    // 8

Key insight: F# functions are curried. A two-argument function is really a function that returns a function. This enables partial application for free.

Tuples & Records

// Tuples: quick grouping
let point = (3, 4)
let (x, y) = point          // destructure

// Records: named fields, immutable by default
type Student = { Name: string; Grade: int }
let alice = { Name = "Alice"; Grade = 92 }
let promoted = { alice with Grade = 95 }   // new copy, alice unchanged

Discriminated Unions

Model data that can be one of several things:

type Shape =
    | Circle of radius: float
    | Rectangle of width: float * height: float

This is more expressive and safer than class hierarchies — the compiler ensures you handle every case.

Pattern Matching & Pipelines

Pattern matching

The Swiss Army knife of F#:

let area shape =
    match shape with
    | Circle r        -> 3.1415 * r * r
    | Rectangle (w,h) -> w * h

If you add a new case to Shape, the compiler warns you everywhere you forgot to handle it. Refactoring becomes fearless.

The pipe operator |>

Transforms nested calls into readable, top-to-bottom pipelines:

List.sum (List.map (fun x -> x * x) (List.filter (fun x -> x % 2 = 0) numbers))

// Easier to read
numbers
|> List.filter (fun x -> x % 2 = 0)
|> List.map (fun x -> x * x)
|> List.sum

Essential list functions

Function Does
List.map Transform each element
List.filter Keep elements matching a condition
List.fold Reduce to a single value
List.sum Sum all elements
List.sort Sort ascending
List.head / List.tail First element / rest
List.length Count elements

For a full reference, see List (FSharp.Core).

F# supports string interpolation using the $ prefix. You can embed any expression inside {}braces:

let name = "Baseload"
let quantity = 50.0m
$"{quantity} MW {name}"   // → "50.0 MW Baseload"

To print to the console, use printfn:

printfn $"{quantity} MW {name}"

Wrap-Up

Key takeaways:

  1. Immutability by default eliminates entire categories of bugs
  2. Records and discriminated unions model real-world data clearly and safely
  3. Pattern matching replaces complex if/else and makes intent explicit
  4. The pipe operator |> makes data transformations readable
  5. Small, focused functions are easy to reuse and compose — as you saw building from Exercise 1 through 4
